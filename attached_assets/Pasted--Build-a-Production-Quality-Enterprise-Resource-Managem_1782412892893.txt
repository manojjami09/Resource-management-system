# Build a Production-Quality Enterprise Resource Management System (RMS) – Frontend Only

Build a **premium SaaS-quality React application** for a **Resource Management System (RMS)** used by IT consulting companies (similar to TCS, Infosys, Accenture, Deloitte, Cognizant, Capgemini).

**IMPORTANT**

* Frontend only.
* Do NOT build any backend.
* Do NOT use Firebase.
* Do NOT use Supabase.
* Do NOT use any database.
* Do NOT use authentication libraries.
* Everything must use **mock API services** and local state.
* I will later connect this UI to my own Java Spring Boot REST API without changing the components.

# Mock API Layer

Never import mock data directly into components.

Instead use

```
services/

employeeService.js
projectService.js
allocationService.js
dashboardService.js
reportService.js
analyticsService.js
```

Example

```
export async function getEmployees(){
   return employees;
}
```

Pretend every function is a REST API.

Later I should only replace

```
return employees
```

with

```
return fetch(...)
```

without touching UI components.
